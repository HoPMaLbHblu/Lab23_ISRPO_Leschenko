# Крестики-нолики

## 1. Описание игры

### Правила игры
Игра «Крестики-нолики» рассчитана на двух игроков.  
Игроки по очереди ставят символы **X** и **O** на поле 3×3.  
Первым ходит игрок **X**.  
Игрок должен поставить свой символ в свободную клетку.

### Логика победы/поражения
Победа определяется, если один игрок собрал три одинаковых символа:
- по горизонтали
- по вертикали
- по диагонали

Если все клетки заполнены и победителя нет — объявляется ничья.  
После победы или ничьи игра останавливается.

### Особенности реализации
- Используется HTML, CSS и JavaScript
- Игровое поле состоит из 9 клеток
- Состояние игры хранится в массиве
- Используются обработчики событий
- DOM обновляется при каждом ходе
- Реализована кнопка перезапуска

---

## 2. Игровая механика

### Как определяется победитель?
После каждого хода вызывается функция `checkResult()`.  
Она проверяет все возможные комбинации победы.  
Если три клетки совпадают и не пустые — объявляется победитель и игра завершается.

### Как обрабатываются клики?
Каждая клетка получает обработчик события `click`.

Алгоритм:
1. Получается индекс клетки
2. Проверяется, свободна ли клетка
3. В массив board записывается символ игрока
4. В клетке отображается символ
5. Вызывается проверка результата

Если игра закончена или клетка занята — клик игнорируется.

### Как работает перезапуск?
При нажатии кнопки вызывается функция `restartGame()`.

Она:
- устанавливает игрока X
- очищает массив board
- включает игру
- очищает клетки
- обновляет статус

## 3. Структура данных

### Объяснить массив board
`let board = ["", "", "", "", "", "", "", "", ""];`
Массив хранит состояние игрового поля.

Индексы:
0 | 1 | 2
3 | 4 | 5
6 | 7 | 8


Значения:
- "X"
- "O"
- "" (пустая клетка)

---

### Объяснить условия победы

const winConditions = [
[0,1,2],
[3,4,5],
[6,7,8],
[0,3,6],
[1,4,7],
[2,5,8],
[0,4,8],
[2,4,6]
];

Это массив всех возможных комбинаций победы.

Каждая строка — это индексы клеток, которые должны совпасть.

---

### Как хранится состояние игры?

Используются переменные:
``` 
currentPlayer
board
gameActive
```

- currentPlayer — текущий игрок (X или O)
- board — состояние поля
- gameActive — активна ли игра

---

## 4. Алгоритмы

### Алгоритм проверки победы

1. Пройтись по массиву winConditions
2. Взять три индекса
3. Проверить совпадение значений
4. Если совпали — победа
5. Остановить игру

---

### Алгоритм определения ничьи

1. Проверить, есть ли пустые клетки
2. Если пустых клеток нет
3. И победитель не найден
4. Объявить ничью

---

### Алгоритм смены игрока

1. Проверить текущего игрока
2. Если X — поставить O
3. Если O — поставить X
4. Обновить статус игрока
